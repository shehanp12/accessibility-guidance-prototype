Frontend

What front end developers need to do about accessibility.

Set page titles

Each page title should be unique, descriptive and concise. It's the first thing a screenreader will read out when the user lands on a page. It's also the link that shows in search results. In both cases a good page title helps users find a page and switch between tabs.

Best practice is generally: Page name | Section – Sitename

For example: Achalasia | Health A to Z – NHS

The first element (here "Achalasia") is the main heading of the page.

If you can, use templates to keep things consistent and use your content management system to generate everything after the pipe symbol.

Define colour contrast

Text contrast

Using colours that contrast well makes it much easier for people to read content comfortably. Ensure the colour contrast between the text colour and background colour meets the following minimum ratio:

Contrast ratio
  • Small text (4.5:1) – text smaller than 24px or 19px if bold.
  • Large text (3:1) – text larger than 24px or 19px if bold.

Check keyboard accessibility

How to test with a keyboard:

  • tab key to progress through links and controls
  • shift-tab to reverse
  • enter to follow links
  • space to select form controls (such as tick boxes)
  • arrows for radio-buttons or tabs
