Testing

Keyboard accessibility is one of the most important parts of accessibility. It enables other input methods, including switches, speech input and screen readers.

The starting point for keyboard accessibility is to use the correct HTML for links, buttons and form controls. If you use components from the service manual, they've been tested for keyboard accessibility.

How to test with a keyboard

How to test with a keyboard:

  • tab key to progress through links and controls
  • shift-tab to reverse
  • enter to follow links
  • space to select form controls (such as tick boxes)
  • arrows for radio-buttons or tabs
