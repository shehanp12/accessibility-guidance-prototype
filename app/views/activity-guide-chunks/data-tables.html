Use tables to show relationships between data

Content

Tables make it easier for users to understand the logical relationships between data across the rows and columns. Only use them when there is a relationship between the "header" cells and the "data" cells in the grid.

Assistive technology uses the relationship to announce the header before each cell and provide column and row information.

View example

You can see an example in the tables component in the service manual.

Screen readers read out the headers and the data together. So the last row would be read out as: "Skin symptoms: itchy blisters. Possible cause: shingles, chickenpox".
