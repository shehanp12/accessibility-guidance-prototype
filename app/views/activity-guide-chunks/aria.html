ARIA patterns

Where components need more descriptions than HTML provides, it needs marking up with an ARIA (Accessibility Rich Internet Applications) pattern.

Simple examples of ARIA used already in the service manual are the Details, Breadcrumbs and Error messages.

How to use ARIA patterns

It is beyond the scope of the service manual to introduce all of ARIA, the W3C note Using ARIA in HTML is a good starting point.

If you have a complex component, check whether it fits into a defined pattern from the ARIA Authoring practices. Patterns that might be useful are:

  • Accordions
  • Modal dialogues
  • Sliders
  • Tabs

If you use an ARIA pattern you will need to include the ARIA attributes as prescribed, and you are taking responsibility for the keyboard interactions as part of the scripting.

Using a complex pattern adds an overhead of testing, both from a technical point of view (does it work as expected with assistive technology), but also from a general usability point of view. You do need to have good evidence that there is a user-need that this component will fulfil.

Our components are well tested, start with those. Another useful resources accessible components is the Inclusive design patterns collection.
