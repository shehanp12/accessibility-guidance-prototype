Set page titles

For: Content, Frontend, Testing

A good page title helps users find and know they're on the right page. It's the link that shows in search results and the first thing a screenreader will read out when the user lands on a page.

Each page title should be unique, descriptive and concise, with important keywords near the beginning.

Best practice

Best practice is generally: Page name | Section – Sitename

For example: Achalasia | Health A to Z – NHS

The first element (here "Achalasia") is the main heading of the page.

If you can, use templates to keep things consistent and use your content management system to generate everything after the pipe symbol.
