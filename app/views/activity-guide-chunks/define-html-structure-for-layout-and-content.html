Define the HTML structure for layout and content

Design

Where a new layout is created it needs to have landmarks defined. The content areas need the headings, lists and data tables defined. These will not necessarily be used as-provided by the developers, but it starts the conversation and helps to prevent inconsistent templates.

How to use landmarks and headings
Landmarks

Landmarks are hidden labels for sections of the page which help users of assistive technology to navigate within a page and build a mental picture of the page structure.

When creating a new layout Define ARIA landmarks to identify regions of a page. It is also a means of allowing users to easily skip over repetitive sections across multiple pages.

  • Banner: Located at the top of the page, usually contains the logo.
  • Navigation – Navigation elements that go to other pages, usually located inside the banner.
  • Search – The search field, usually located inside the banner.
  • Main – Contains the main content of the page, it should be unique on each page.
  • ContentInfo – The footer, this is shared across pages.

Headings

Where headings are used around the layout of the page, the levels of those headings needs to be considered and defined.

Headings are labels for the content of a page (more granular than landmarks), and should title the content that follows them.
