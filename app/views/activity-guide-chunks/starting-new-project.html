Ask key questions at the start of a project

Product and delivery

If the answer to any of these questions is 'yes', it's a sign that the project will need a higher level of accessibility expertise or support.

Key questions
Does the project affect people with disabilities differently?

For example: Does the user's journey include physical access to a GP surgery? If so, include people with disabilities in any user research.

Will the project use any technologies you haven't already tested for accessibility?

If so, you will need to test them, and the team working on them will need a higher level of accessibility knowledge or support.

Will the interface need new components?

You will need to make sure they are accessible and to test them.

Will there be a different colour scheme?

The colour combinations will need testing, both for text and graphical contrast.

Will the project use multimedia PDFs, infographics, large data tables or other less common content?

If so, check the capabilities of the system and the people creating them so that you can be sure that they will be accessible.

Is there a reason to include 'Easy Read' documents or translations?

If there is some key information that everyone must understand (such as a guide to start using GP online services), then consider including an Easy Read version for people with low-literacy or learning disabilities. Or you could link to Easy Read content from other organisations.
