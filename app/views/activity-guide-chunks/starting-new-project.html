Ask questions at the start of the project

Product and delivery

Ask yourself how your project will affect people with access needs, how much work you will need to do to make it accessible and whether your team has the capacity and expertise to do it.

Questions to ask

If the answer to any of these questions is 'yes', it's a sign that the project will need a higher level of accessibility expertise or support.

Does the project affect people with disabilities differently?

For example, does the user's journey include physical access to a GP surgery? If so, include people with disabilities in any user research.

Will the project use any technologies you haven't already tested for accessibility?

You will need to test them, and the team working on them will need a higher level of accessibility knowledge or support.

Will the interface need new components?

You'll need to make sure they are accessible and test them.

Will there be a different colour scheme?

You'll need to test the colour combinations, both for text and graphical contrast.

Will the project use multimedia, PDFs, infographics, large data tables or other less common content?

Check the capabilities of the systems and the people creating them so that you can be sure that they will be accessible.

Is there a reason to include 'Easy Read' documents or translations?

If there is some key information that everyone must understand (such as a guide to start using a new national service), then consider including an Easy Read version for people with low-literacy or learning disabilities. Or you could link to Easy Read content from other organisations.
