Use alternative text for functional images

For: Frontend, Testing

Some images are functional, like an action link. If people can't see the image, they need an alternative in the form of "alt‑text".

Images with text labels

If the image has a text label that conveys its meaning, the alt‑text should be "null". The attribute should be present, but empty.

Examples

An example would be a link to a health app which includes a brand image and the name of the app. The app name says the same as the brand image, so the image doesn't need explaining.

For a standard image, like a JPG photo, use:

<img src="image.jpg" alt="">

For an inline SVG image, like an icon , use:

<svg aria-hidden="true">.

Image that don't have text

You must include alt-text. For example, the play button on a media player or carousel:

<img src="play.jpg" alt="Play">

.[INSERT PLAY BUTTON IMAGE]

Examples

For inline SVG images, use:

<svg ...>Play

The alt text (or SVG title) of a functional image should say what it does, rather than what it looks like. For example, "Play" for a video, not "Triangle", and "Search", not "Magnifying glass".
