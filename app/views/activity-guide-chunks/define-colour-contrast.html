Define colour contrast

Design
Frontend

It's easier for people to read and interact with content if you use colours that contrast well. Where possible, the NHS aims for a higher level of contrast (AAA).

Use a tool to check the colour contrast:

  • Web Aim's colour contrast checker
  • Color Oracle (free colour blindness simulator)
Text contrast

Make sure the contrast between the text colour and background colour meets the following minimum ratios (AAA):

  • small text (7:1) – text smaller than 24px or 19px if bold
  • large text (4.5:1) – text larger than 24px or 19px if bold
Non-text contrast

People with limited vision need to be able to identify and understand:

  • user interface components (like text inputs and radio buttons)
  • meaningful graphics (like icons)

To check them:

  • identify the elements that convey the meaning of the component or graphic
  • use a tool to calculate the ratio between the graphical element and the adjacent colour - for example, the white arrow (foreground) and the green background in the action link example below
  • check the colour contrast meets the minimum ratio of 3:1

If a graphic, such as an icon, is essential to understand the content, it must meet the minimum contrast ratio. If it also has text, it doesn't have to meet the requirement but it is still recommended.

Components that are visible but not currently active (like a submit button that isn't active until you've filled in the form) are exempt from the requirement, but if you can meet the minimum contrast without it being confusing, that will help people with low vision.
