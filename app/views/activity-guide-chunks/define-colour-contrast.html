Define colour contrast

Design
Frontend

It's easier for people to read and interact with content if you use colours that contrast well. Where possible, the NHS aims to meet the higher level (AAA) requirements for contrast.

This means that the contrast between the text colour and background colour should be a minimum of:

  • 7:1 – for small text (smaller than 24px or 19px if bold)
  • 4.5:1 – for large text (larger than 24px or 19px if bold)

For components (like text inputs) or graphics (like icons), the contrast between the graphical element and the adjacent colour should be a minimum ratio of 4.5:1.

Tools to test colour contrast

Use a tool to check the colour contrast:

  • Web Aim's colour contrast checker
  • Color Oracle (free colour blindness simulator)
More about components and graphics

Use a tool to calculate the ratio between the graphical element and the adjacent colour - for example, the white arrow (foreground) and the green background in an action link/p>

If a graphic, such as an icon, is essential to understand the content, it must meet the minimum contrast ratio. If it also has text, it doesn't have to meet the requirement but this is still recommended.

Components that are visible but not currently active (like a submit button that isn't active until you've filled in the form) are exempt from the requirement, but if you can meet the minimum contrast without it being confusing, that will help people with low vision.
