Key questions for new projects

If the answer to any the questions below is ‘yes’, then it is a sign the project will need a higher level of accessibility expertise or support.

View key questions
Does the project impact people with disabilities differently?

For example: Does the user's journey include physical access to a GP surgery? In which case include people with disabilities in any user research.

Will the project be using new technologies? (Rather than the standard CMS and components)

If so, the capabilities of the technology need to be assessed, and the team working on it will need a higher level of accessibility knowledge or support.

Will the interface require new components?

These would need work to make accessible and be tested to ensure they are accessible.

Will there be a different colour scheme?

These colour combinations would need testing, both for text and graphical contrast.

Will the project use multimedia PDFs, infographics, large data tables or other less common content?

If so, check the capabilities of the system and the people creating them. NHS UK has experience making these types of content accessible, but it does not happen by default.

Is there a reason to include 'easy read' documents?

If there is some key information that everyone must understand (such as the guide to start using GP online services), then consider including an easy read version for people with low-literacy or learning disabilities.
